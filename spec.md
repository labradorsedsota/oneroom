# Project OneRoom - Level Design Spec (Phase 1)

## L1: The Button (按钮)
*   **提示 (Hint):** "Under Pressure"
*   **玩法 (Gameplay):** 玩家控制角色，从左边跳到中间的按钮上，压住它。
*   **预期体验 (Exp):** 建立最基础的物理互动认知。明白“我有重量，按钮需要被踩”。
*   **用户操作 (User):** 按 `D` (右移) -> 按 `Space` (跳跃) -> 松开 `Space` (落地)。
*   **角色动作 (Char):** 向右跑动 -> 起跳 -> 落在按钮上方 -> 压扁按钮。
*   **反馈 (Feedback):** 按钮发出沉闷的“咔哒”声，变红/发光。右边的门缓缓滑开。
*   **设计意图 (Intent):** **Tutorial (教学)**。教会移动、跳跃、交互。

## L2: The Decoy (诱饵)
*   **提示 (Hint):** "Ignore It"
*   **玩法 (Gameplay):** 场景和 L1 一样，但这次不能踩按钮。必须直接走到门口。
*   **预期体验 (Exp):** 产生困惑 -> 尝试踩按钮失败 -> 发现原来什么都不做才是对的。
*   **用户操作 (User):** 按 `D` (右移) -> 按 `Space` (跳过按钮) -> 继续按 `D` 走到门口。
*   **角色动作 (Char):** 向右跑 -> 跳过中间的凸起物（按钮） -> 停在门前。
*   **反馈 (Feedback):** 只要没踩按钮，门自动感应开启。如果踩了，门变红锁死，发出错误音效。
*   **设计意图 (Intent):** **Subversion (颠覆)**。打破 L1 建立的规则，训练抑制冲动。

## L3: Patience (耐心)
*   **提示 (Hint):** "Hold Your Breath"
*   **玩法 (Gameplay):** 进关卡后，什么都不做，原地站立 5 秒。
*   **预期体验 (Exp):** 焦虑 -> 忍耐 -> 释然。
*   **用户操作 (User):** **松开双手**。没有任何输入。
*   **角色动作 (Char):** 原地呼吸动画（Idle）。如果动了，会有急促的喘息声或心跳加速声。
*   **反馈 (Feedback):** 背景音里传来心跳声（咚...咚...）。5 秒后，一声清脆的“叮”，门开。
*   **设计意图 (Intent):** **Inaction (无为)**。将“不操作”作为一种操作。

## L4: The Mouse (上帝之手)
*   **提示 (Hint):** "Hands On"
*   **玩法 (Gameplay):** 角色踩按钮无效。必须用鼠标点击屏幕上的按钮。
*   **预期体验 (Exp):** 意识到“我”不仅是角色，还是屏幕前的玩家。
*   **用户操作 (User):** 移动鼠标 -> 左键点击屏幕中央的红色圆形。
*   **角色动作 (Char):** 角色站在旁边看着（Idle）。按钮被看不见的手按下。
*   **反馈 (Feedback):** 鼠标点击瞬间，按钮发光并下陷，门开。
*   **设计意图 (Intent):** **Breaking the 4th Wall (打破第四面墙)**。引入鼠标交互。

## L5: Keep Jumping (一直跳)
*   **提示 (Hint):** "Don't Stop"
*   **玩法 (Gameplay):** 在按钮上连续快速跳跃，像给气球充气一样把门“充”开。
*   **预期体验 (Exp):** 紧张感、节奏感。
*   **用户操作 (User):** 在按钮区域内，有节奏地按 `Space` -> `Space` -> `Space` (至少 3-5 次)。
*   **角色动作 (Char):** 角色在原地连续起跳、落下。每次落下，按钮都被压得更扁一点。
*   **反馈 (Feedback):** 每次落地都有音效叠加（Do-Re-Mi...），门上的指示灯一格格亮起。充满后，门弹开。
*   **设计意图 (Intent):** **Action & Rhythm (动作与节奏)**。引入频率控制。

# Project OneRoom - Level Design Spec (Phase 2)

## L6: Heavy (沉重)
*   **提示 (Hint):** "Carrying the Weight"
*   **玩法 (Gameplay):** 角色的重力 (Gravity Scale) 增加 3 倍，移动速度 (Speed) 减半。
*   **预期体验 (Exp):** 感觉角色非常笨重，跳跃高度极低，行动迟缓。
*   **用户操作 (User):** 按 `D` (右移) -> 必须精准助跑或利用边缘跳跃，否则很容易掉进坑里或者够不着按钮。
*   **角色动作 (Char):** 走路姿势沉重，落地时没有弹性，像铁块一样砸在地上。
*   **反馈 (Feedback):** 落地时发出沉闷的金属撞击声。屏幕轻微震动。
*   **设计意图 (Intent):** **Physics Parameter (物理参数)**。感知重力变化。

## L7: Anti-Gravity (反重力)
*   **提示 (Hint):** "Upside Down"
*   **玩法 (Gameplay):** 进关卡瞬间重力自动翻转（-980）。角色被吸到天花板上。按钮和门都在天花板上。
*   **预期体验 (Exp):** 空间错乱。虽然操作没变，但大脑需要适应上下颠倒的视觉。
*   **用户操作 (User):** 按 `D` (右移) -> 角色向**屏幕右边**移动（哪怕它是头朝下的）。按 `Space` (跳跃) -> 角色向**屏幕下方**跳离天花板。
*   **角色动作 (Char):** 头朝下脚朝上，贴着天花板跑动。
*   **反馈 (Feedback):** 所有的灰尘粒子都往上飘。落地音效从头顶传来。
*   **设计意图 (Intent):** **Inversion (反转)**。纯视觉欺骗，操作逻辑（左右）保持绝对坐标不变。

## L8: Slippery (打滑)
*   **提示 (Hint):** "Ice Floor"
*   **玩法 (Gameplay):** 地面摩擦力极低 (Friction ≈ 0.05)。**禁用跳跃 (Jump Disabled)**。
*   **预期体验 (Exp):** 像在冰面上。必须极其小心地控制速度，否则会滑过头。
*   **用户操作 (User):** 轻点 `D` 起步 -> 提前反向按 `A` 刹车 -> 必须**完全停在**按钮区域内（Velocity ≈ 0）。
*   **角色动作 (Char):** 脚底打滑，身体倾斜以保持平衡。
*   **反馈 (Feedback):** 摩擦时有尖锐的“呲溜”声。角色脚下有冰屑粒子。
*   **设计意图 (Intent):** **Deceleration & Precision (减速与精准)**。考验微操与预判。

## L9: The Box (隐形机关)
*   **提示 (Hint):** "Step Up"
*   **玩法 (Gameplay):** 初始场景空旷。在按钮和门之间有一个**隐形触发点**。踩中后，地面升起一个箱子。
*   **预期体验 (Exp):** 惊喜感（Surprise）与工具发现。
*   **用户操作 (User):** 走到触发点 -> 被升起的箱子顶高 -> 从箱子上跳下来 -> 把箱子推到按钮上 -> 门开。
*   **角色动作 (Char):** 被顶起，落地，做出推挤动作。
*   **反馈 (Feedback):** 箱子升起时的机械声。推箱子摩擦地面的声音。
*   **设计意图 (Intent):** **Hidden Trigger & Object Interaction (隐形机关与物体交互)**。

## L10: Invisible (盲人摸象)
*   **提示 (Hint):** "Trust Your Gut"
*   **玩法 (Gameplay):** 地形完全不可见（隐形 TileMap）。只有角色和门发光。
*   **预期体验 (Exp):** 恐惧与探索。完全依赖听觉和试错。
*   **用户操作 (User):** 试探性跳跃 -> 撞墙/落地（听声音判断位置） -> 记住地形结构 -> 盲跳到按钮位置。
*   **角色动作 (Char):** 正常的跳跃跑动，但在黑暗中显得很孤独。
*   **反馈 (Feedback):** 每次落地、撞墙都有明显的回声。角色周围有一圈微弱的光晕照亮脚下一点点路。
*   **设计意图 (Intent):** **Information Deprivation (信息剥夺)**。感官挑战。

# Project OneRoom - Level Design Spec (Phase 3)

## L11: The UI (界面)
*   **提示 (Hint):** "Look Closer"
*   **玩法 (Gameplay):** 场景中的按钮无论怎么踩都没有反应。真正的按钮伪装成 HUD 上的一个 UI 元素（比如左上角的"⚙️"设置图标）。必须用鼠标点击它。
*   **预期体验 (Exp):** 困惑 -> 反复尝试场景内按钮 -> 注意到 HUD 上有异常 -> 恍然大悟。
*   **用户操作 (User):** 尝试踩按钮无效 -> 移动鼠标到 HUD 区域 -> 左键点击伪装的 UI 元素。
*   **角色动作 (Char):** 角色站在无效按钮旁（Idle），看着上方（暗示方向）。
*   **反馈 (Feedback):** 点击 UI 元素后，整个 HUD 碎裂/消散的动画，露出门。发出玻璃破碎音效。
*   **设计意图 (Intent):** **UI as Gameplay (界面即玩法)**。打破"HUD 是安全区"的认知，与 L4（鼠标）形成进阶呼应。

## L12: The Wall (穿墙)
*   **提示 (Hint):** "No Walls"
*   **玩法 (Gameplay):** 按钮在屏幕最右侧，角色在最左侧，中间有一堵不可逾越的高墙。走出屏幕左边界会从右边界出现（Screen Wrap）。
*   **预期体验 (Exp):** 尝试翻墙失败 -> 走投无路 -> 走到屏幕边缘 -> 发现空间循环 -> 从另一侧绕到按钮。
*   **用户操作 (User):** 按 `A`（左移）走出屏幕左边界 -> 从右边界出现 -> 按 `D` 走到按钮 -> 踩下。
*   **角色动作 (Char):** 角色走到屏幕边缘时身体逐渐消失，同时从另一侧逐渐出现。
*   **反馈 (Feedback):** 穿越边界时有空间扭曲的音效（类似 Portal）。屏幕边缘有微弱的光带提示可穿越。
*   **设计意图 (Intent):** **Spatial Wrap (空间循环)**。打破"屏幕边界是墙"的认知。

## L13: Time Travel (时间)
*   **提示 (Hint):** "Tick Tock"
*   **玩法 (Gameplay):** 场景中有一个可见的大时钟（显示真实秒数）。按钮只在偶数秒处于可激活状态（亮起），奇数秒锁定（熄灭）。必须在偶数秒踩上按钮。
*   **预期体验 (Exp):** 观察时钟 -> 发现按钮闪烁规律 -> 计算时机 -> 精准踩踏。
*   **用户操作 (User):** 观察时钟和按钮闪烁规律 -> 在偶数秒瞬间按 `D` + `Space` 跳到按钮上。
*   **角色动作 (Char):** 正常跑跳。踩中激活态按钮时稳稳落地；踩中锁定态按钮时被弹开。
*   **反馈 (Feedback):** 时钟滴答声贯穿始终。偶数秒按钮亮起发出柔和光芒，奇数秒熄灭变灰。成功时时钟指针停转，发出钟声。
*   **设计意图 (Intent):** **Real-time Constraint (实时约束)**。将现实世界的时间引入游戏规则。

## L14: Keyboard (键盘)
*   **提示 (Hint):** "Say It"
*   **玩法 (Gameplay):** 场景中门上刻着四个乱序字母（比如 "P-N-E-O"）。没有按钮。玩家必须用键盘按正确顺序打出 "OPEN"。输入不可见，输错则重置。
*   **预期体验 (Exp):** 困惑（没有按钮？）-> 注意到门上的字母 -> 尝试键盘输入 -> 试错 -> 拼出 OPEN。
*   **用户操作 (User):** 依次按键盘 `O` -> `P` -> `E` -> `N`。方向键/空格不起作用。
*   **角色动作 (Char):** 角色面朝门站立（Idle）。每按对一个字母，角色做出一个"比划"动作，门上对应字母亮起。
*   **反馈 (Feedback):** 每个正确字母伴随一个上升音阶（Do-Re-Mi-Fa）。全部正确后四个字母重排为 "OPEN" 并发光，门开。按错字母发出低沉错误音，所有字母熄灭重置。
*   **设计意图 (Intent):** **Keyboard as Puzzle (键盘即谜题)**。键盘不再是移动工具，而是直接的解谜媒介。

## L15: The End? (终点)
*   **提示 (Hint):** "Game Over"
*   **玩法 (Gameplay):** 进入关卡后，没有按钮，没有门。屏幕中央弹出一个大大的 "QUIT" 按钮和一行字："感谢游玩"。点击 QUIT 后播放假的退出动画（黑屏 + 制作人名单），然后突然画面碎裂，露出真正的隐藏通道在屏幕底部。
*   **预期体验 (Exp):** 失落（结束了？）-> 点击退出 -> 被骗 -> 惊喜 -> 找到真正的终点。
*   **用户操作 (User):** 鼠标点击 "QUIT" 按钮 -> 观看假结局 -> 画面碎裂后按 `S`（下移）或 `Space`（跳入）进入底部通道。
*   **角色动作 (Char):** 假结局期间角色消失。画面碎裂后角色重新出现，站在裂缝边缘，向下看。
*   **反馈 (Feedback):** 假结局：舒缓的钢琴曲 + 滚动字幕。碎裂：玻璃破碎音 + 画面裂纹特效。进入隐藏通道后，明亮温暖的光芒，暗示"真正的结局"。
*   **设计意图 (Intent):** **Fake Ending (假结局)**。质疑"结束"的定义，为整个游戏画上一个意味深长的句号。
